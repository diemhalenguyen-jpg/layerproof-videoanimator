How-To Guide

Reveal Builder

A step-by-step reference for building progressive slide reveals synced to a voiceover — from uploading your images to presenting a polished, animated sequence.

🎬
01

Overview

Reveal Builder lets you take a set of slide images and define exactly which parts of each image appear — and when — during a video presentation. Instead of the full slide appearing at once, elements emerge progressively as the speaker talks about them.

The core idea is simple: you draw rectangular reveal regions on each slide. During the presentation, those regions are uncovered one by one, triggered either manually (with the spacebar) or automatically by audio timestamps from a voiceover recording.

What you create

A scripted, timed sequence of slide reveals — each region appearing in sync with your narration.

What you need

Your slide images (PNG or JPG), and optionally a voiceover audio file and a free Gemini API key for auto-transcription.

Projects are saved as a JSON file that preserves all your regions, timings, and settings — so you can pick up where you left off at any time.

02

Getting Started

When you open the tool, you land on the upload screen. This is where you load your slide images and choose your workflow mode.

Manual mode

You draw reveal regions freely on each slide with no pre-set order. Best when you are building the reveal from scratch or don't have a written script.

Script mode

You paste a script containing special tags like [S2: phrase]. The tool uses these to guide you through drawing regions in the right order, matched to transcript cues.

1
Choose Manual or Script mode using the toggle buttons at the top of the upload screen.
2
Drag and drop your slide images onto the dropzone, or click it to browse. Select all slides at once — they will be sorted by filename. Name them 1.png, 2.png, etc. for best results.
3
If using Script mode, paste your script into the text area that appears below the dropzone.
4
Click Load → to enter the editor.
Tip: Images are displayed in a 9:16 (vertical/portrait) aspect ratio — the standard format for short-form video. If your slides are in 16:9 landscape, you can set a background color to fill the bars above and below.

Once in the editor, you will see a filmstrip on the left, a central canvas, a Reveal Order panel on the right, and a settings panel on the far right. You can add more images at any time using the + Add images button at the bottom of the filmstrip.

✏️
03

Drawing Reveal Regions

The canvas in the center of the editor shows the currently selected slide. To define what will be revealed, you draw rectangles directly on the image.

1
Click a slide in the filmstrip on the left to select it.
2
Click and drag on the canvas to draw a rectangle around the element you want to reveal. Release the mouse to confirm.
3
A numbered box appears in the Reveal Order panel on the right. Each box represents one reveal step for that slide.
4
Repeat for as many regions as you need. Regions are revealed in the numbered order shown in the panel.
Region coverage: Each region covers the area you draw. During the presentation, the slide starts fully hidden and regions are uncovered in sequence — so draw regions around every part of the slide you want to show. If you want the entire slide visible by the end, make sure every element has its own region (or draw one large region that covers the full slide).

The Margin slider in the top toolbar adds a small buffer around each drawn region. A margin of 5% (the default) means each reveal box will be slightly larger than what you drew, giving a bit of breathing room around text and icons.

Use Ctrl+Z to undo the last drawn region, and Ctrl+Y to redo. You can also use the Undo / Redo buttons at the bottom of the right panel.

To remove all regions for the current slide, click Reset crops in the top toolbar.

04

Script Mode

Script mode is a guided workflow for building reveals that map precisely to a written script. You annotate the script with tags that tell the tool which phrase in the transcript should trigger each reveal region.

The tag format

Tags use this format:

[S2: phrase to search for]

Where S2 refers to slide 2 (matching the image filename 2.png), and phrase to search for is the exact wording from your transcript that should trigger the reveal on that slide.

A full script excerpt might look like:

Brand recall is [S2: brand recall is] the most important metric... A strong identity [S3: a strong identity] drives consistent growth.
How it works: The tool parses your script to extract an ordered list of tags. As you work through each slide, the tag queue panel (visible in the Reveal Order area) shows you exactly which region to draw next and which phrase it corresponds to. Once you draw the region, the tool automatically assigns the matching transcript cue as the trigger time.
1
On the upload screen, select Script mode and paste your annotated script into the text area.
2
Load your images. The tag queue will appear in the Reveal Order panel, showing the first tag you need to draw.
3
Navigate to the slide indicated by the tag and draw the region around the element it refers to. The queue advances automatically.
4
Continue through all tags. Completed tags show a green checkmark; the next pending tag is highlighted in amber.
Best practice: Write your script first, then listen through the voiceover and note which phrases correspond to key moments on each slide. Annotate those moments with tags before loading the tool.

05

Transcription

The tool can automatically transcribe your voiceover audio into timestamped segments using Google's Gemini AI. Each segment becomes a clickable cue you can assign to a reveal region.

1
Get a Gemini API key. Visit aistudio.google.com, sign in with a Google account, and create a free API key.
2
Enter the key in the Gemini key field in the top toolbar. Click the save button (💾) next to it — the key is stored in your browser so you don't need to re-enter it each session.
3
Upload your audio using the ♪ Audio button in the toolbar. Accepted formats include MP3, WAV, M4A, and most common audio types.
4
Click Transcribe (⚡ Transcribe). The tool sends the audio to Gemini and receives a word-by-word, timestamped transcript. This takes 10–60 seconds depending on audio length.
5
The transcript appears in the TRANSCRIPT panel as a list of segments, each showing a timestamp and the spoken text.

Using transcript segments

Click any segment in the transcript list to assign it as the trigger cue for the currently selected reveal region. The region will be revealed at that moment in the audio during the presentation.

You can also click directly on the text of any segment to edit it. This is useful for correcting transcription errors that would otherwise show up incorrectly in captions.

Auto-matching in Script mode: When you have both a transcript and script tags loaded, the tool can attempt to automatically match each tag phrase to the closest segment in the transcript. Look for the Auto-match button in the Reveal Order panel.
⬆️
06

Reveal Order

The Reveal Order panel (right side of the editor) shows all the reveal regions for the current slide, numbered in the order they will be uncovered during the presentation.

Reordering regions

Drag any region card up or down in the list to change the reveal order. The numbers update automatically.

Deleting a region

Click the delete button (×) on the right side of a region card to remove it. Use Undo if you remove one by mistake.

Editing the cue time

Click a region card to select it. The card expands to show a keyword/cue input field. You can either:

A
Click a transcript segment — if a transcript is loaded, click any segment row in the TRANSCRIPT panel. The timestamp from that segment is assigned as the trigger for the selected region.
B
Type a keyword — enter a word or phrase in the keyword field. During auto-reveal, the tool will search the transcript for that phrase and trigger the reveal at the matching timestamp.
No cue set? Regions without a cue time or keyword can still be revealed manually during the presentation using the spacebar or arrow keys. Only auto-reveal requires a cue.

The small badge on each region card shows its status: green means the cue is matched to a transcript segment, amber means the keyword hasn't been matched yet, and grey means no cue is set.

07

Cats

Cat stickers are animated images that appear on the canvas each time a new reveal region is uncovered. They appear at the position of the most recently revealed region and animate in, adding a playful visual flourish to each reveal step.

1
In the right panel, find the CATS section. Click 📁 Load to open a folder picker.
2
Select a folder containing PNG or GIF image files. All images in the folder will be loaded as available stickers.
3
The loaded images appear as small thumbnails in the panel. Click one to select it as the active sticker, or leave the default to cycle through all loaded images.
4
Use the % size input to control how large the cat sticker appears, as a percentage of the canvas height. The default is 17%.

During the presentation, you can click and drag the cat sticker to reposition it on the canvas if it is obscuring something important. The cat's position is remembered between reveals until you move it again.

Each region card in the Reveal Order panel also has a small cat thumbnail area. You can assign a specific sticker to a specific region by clicking the swatch there — useful if you want different cats for different slides.

Reload button: If you update the images in your cat folder, click the ↺ button next to Load to reload them without having to pick the folder again.

08

Image Adjustments

The IMAGE section of the right panel provides two non-destructive adjustments that are applied to the image during both the preview and the presentation. Your original file is never modified.

Exposure

Brightens or darkens the entire image. Drag right to brighten, left to darken. Useful if your screenshots look too dark or washed-out on screen.

Blacks

Crushes dark tones toward pure black. Drag right to deepen the shadows. This is especially effective for slides with a dark background that appears slightly grey — it makes the background look properly black and removes shadow artefacts.

These adjustments apply to the current slide only. Each slide in the filmstrip can have its own independent exposure and blacks settings.

Workflow tip: If you're working with slides exported from a presentation app, try setting Blacks to 15–25 to clean up any compression artefacts in the dark areas before drawing your regions.

09

Background Color

The BACKGROUND section controls the fill color that appears around the slide image when the image does not fully cover the 9:16 canvas area.

This is most relevant for landscape (16:9) slides, where black bars appear above and below the image. Matching the background color to the slide's own background makes the overall frame look intentional and seamless.

1
Click the color swatch to open the color picker and choose any color.
2
Use the preset swatches to quickly select common slide backgrounds — black, white, and a few greys are available by default.
3
The hex value next to the swatch shows the current color. This updates live as you change the color.
Most common use case: If your slides have a pure black background, set the background to #000000. The image will appear to fill the frame edge-to-edge visually, even though it is technically letterboxed.

Highlights

Highlights are colored rectangular boxes drawn on the slide that appear and disappear at specific timestamps during the presentation. They are distinct from reveal regions — highlights overlay the slide as a visual annotation (like a callout box), rather than uncovering content.

Drawing a highlight

1
In the bottom of the right panel, find the HIGHLIGHTS section. Click the ⬛ Draw button to enter highlight draw mode. The button turns red when active.
2
Optionally, choose a highlight color using the color swatch that appears, and adjust the line width with the small slider.
3
Click and drag on the canvas to draw the highlight box. Release to confirm. The highlight appears in the list below.
4
Click ⬛ Draw again to exit draw mode and return to region drawing.

Setting show and hide times

Each highlight in the list has two time buttons: Show at and Hide at.

1
Click the Show at button on a highlight item. It turns amber, indicating it is waiting for you to pick a time.
2
Click any segment in the TRANSCRIPT panel. That segment's timestamp becomes the "show at" time for the highlight.
3
Repeat with Hide at to set when the highlight disappears. If no hide time is set, the highlight remains visible for the rest of the slide.

Captions

When a transcript is loaded and audio is playing, the tool can display the spoken words as live subtitles on the canvas. This is the CAPTION feature, controlled from the right panel.

CC Toggle

The Show captions checkbox turns captions on or off. This is also mirrored by the CC checkbox in the presentation toolbar.

Font size

The Font size slider sets the caption text size in canvas points. Larger slides benefit from a larger size (24–36); smaller previews look cleaner at 14–18.

The Position slider moves the caption text up or down within the frame. The value is expressed as a percentage from the bottom — 8% means the text sits just above the bottom edge. Increase it to push the text higher if it overlaps with content in the lower part of the slide.

Editing captions: Caption text comes directly from your transcript segments. If you notice a transcription error in the captions during playback, go to the TRANSCRIPT panel and click on the segment text to edit it inline.

12

Presentation

When you're ready to present, click the Present → button in the top-right toolbar. This switches to a full-screen presentation view with a clean black background and a minimal control bar at the top.

Navigation controls

Key Action
Space Next reveal — uncovers the next region on the current slide
→ or ↓ Next reveal — same as Space
← or ↑ Previous reveal — steps back one reveal
N Next slide — moves to the next slide in the sequence
P Previous slide — moves to the previous slide

Audio playback

If you uploaded a voiceover audio file, the audio controls appear in the top-right of the presentation bar. Press the play button or click anywhere on the canvas to play or pause the audio.

Auto-reveal

When audio is playing and your reveal regions have assigned cue times, the tool automatically uncovers each region at the correct moment in the audio. You do not need to press anything — the reveals happen in sync with the narration.

Combining manual and auto: Even with auto-reveal active, you can still press Space to advance manually. If you press Space before an auto-reveal fires, it will trigger that reveal immediately.

The dot indicators at the top-left of the presentation bar show your progress through the slides. A green dot means the slide is complete; amber indicates the current slide.

Click ← Edit in the top-left to return to the editor at any time.

Saving and Loading

All your work — reveal regions, cue times, image adjustments, highlights, cat assignments, and settings — is saved in a project file. The project file is a standard JSON file that you keep alongside your slide images.

Saving a project

Click the 💾 Save button in the top toolbar. Your browser will download a .json file. Name it something descriptive and keep it in the same folder as your slide images.

You can also save from the Reveal Order panel using the small 💾 icon at the top of that panel.

Loading a saved project

Click the 📂 Open button in the toolbar or the matching icon in the Reveal Order panel. Browse to your .json file and select it. The project will reload with all your settings restored.

Important: The project file stores references to your images by filename. When re-opening a project, load your images first (using the upload screen), then open the project file. If the image filenames match, everything will align correctly.
Audio is not saved in the project file — it would make the file too large. After re-opening a project, simply re-upload the audio file using the ♪ Audio button.

Save frequently

Save your project often, especially after setting up cue times or making changes to the reveal order. The tool has no auto-save — closing the browser tab will lose unsaved work.

Workflow tips

★
Prepare images first. Export slides as individual PNG files numbered 1.png, 2.png, etc. Use the highest resolution your presentation app supports.
★
Transcribe before drawing regions. Having the transcript loaded before you draw regions means you can immediately assign cue times as you go, rather than having to go back and set them all afterwards.
★
Use Blacks aggressively on dark slides. A Blacks value of 20–30 on dark-background slides significantly improves how they look on screen by eliminating grey cast and compression noise.
★
Keep region overlap minimal. If two reveal regions on the same slide overlap heavily, the reveal effect can look cluttered. Draw regions tightly around individual text blocks, icons, or chart elements rather than large areas.
★
Add a "full reveal" region last. As a final region on each slide, draw a box that covers the entire canvas. This ensures the slide is fully visible by the end of that segment, even if earlier regions didn't cover everything.
★
Test in presentation mode often. Switch to Present mode after setting up each slide to check that the reveal timing and order look right before moving on.
★
Save the project JSON next to your images. Keeping the JSON file in the same folder as your slide images makes it easier to reload the project and share it with others.
Recording the presentation: Use a screen recorder (like QuickTime on Mac or OBS) to capture the presentation mode while the audio plays back. The result is a fully produced reveal video that you can publish or edit further.
